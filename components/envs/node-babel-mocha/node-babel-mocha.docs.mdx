An env for Node components that use Babel as the compiler and Mocha as the tester.
This env will:

- use babel to compile the code to cjs
- use TS for d.ts generation
- use Mocha for testing

The directory "excluded-fixtures" is excluded from compilation. Babel will not try to compile it.
Instead, the files there will be copied to the dist dir.
It's also excluded from the tests, the spec files there are not set as dev-files.
Lastly, the tsconfig has it as excluded so we don't type-check it.
